# Contributing to Soughbreighittee

Thank you for your interest in contributing to Soughbreighittee! This project aims to provide comprehensive, harm-reduction focused information about Opiate Use Disorder recovery methods.

## 🎯 Project Mission

To create the most comprehensive, accurate, and accessible resource for understanding all available recovery methods for Opiate Use Disorder, with a focus on harm reduction and informed choice.

## 🤝 How to Contribute

### Adding or Updating Recovery Methods

We welcome contributions that:
- Add new recovery methods to the database
- Update existing method information with new research or data
- Improve descriptions and explanations
- Add resources and references

When adding or updating methods, please ensure:
- Information is factually accurate and well-sourced
- Both benefits and risks are clearly presented
- The safety level is appropriately assessed
- Evidence level accurately reflects scientific support
- All required fields are completed

### Code Contributions

1. **Fork the repository** and create a new branch for your changes
2. **Follow the existing code style**:
   - Use PEP 8 for Python code
   - Add type hints to function signatures
   - Include docstrings for all functions and classes
   - Use meaningful variable names
3. **Test your changes**:
   ```bash
   python tests/test_basic.py
   ```
4. **Commit with clear messages** that explain what and why
5. **Submit a pull request** with a description of your changes

### Documentation Improvements

- Fix typos or unclear explanations
- Add examples and usage scenarios
- Improve installation or setup instructions
- Translate documentation to other languages

### Reporting Issues

When reporting bugs or suggesting features:
- Check if the issue already exists
- Provide clear, detailed descriptions
- Include steps to reproduce (for bugs)
- Suggest potential solutions if possible

## 📋 Development Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/VpkDevs/soughbreighittee.git
   cd soughbreighittee
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run tests:
   ```bash
   python tests/test_basic.py
   ```

4. Test the CLI:
   ```bash
   python main.py --help
   ```

## 🧪 Testing Guidelines

- Add tests for new functionality
- Ensure all existing tests pass
- Test edge cases and error conditions
- Verify changes work across different platforms

## 📝 Code Style

- Follow PEP 8 style guidelines
- Use type hints for function parameters and return values
- Add docstrings to all public functions and classes
- Keep functions focused and single-purpose
- Use constants for magic numbers
- Handle errors gracefully with helpful messages

## 🔒 Sensitive Content Guidelines

This project deals with addiction and recovery, which are sensitive topics:
- Use respectful, non-stigmatizing language
- Avoid judgment or moralizing
- Focus on factual, evidence-based information
- Respect privacy and confidentiality
- Prioritize harm reduction and safety

## ✅ Pull Request Checklist

Before submitting a pull request:
- [ ] Code follows the style guidelines
- [ ] All tests pass
- [ ] New code has tests (if applicable)
- [ ] Documentation is updated (if applicable)
- [ ] Commit messages are clear and descriptive
- [ ] Changes are focused and minimal

## 🤔 Questions?

If you have questions about contributing:
- Check existing issues and discussions
- Open a new issue with your question
- Be patient and respectful
